# applippli-key サイト 開発・再開手順

このサイト（key.applippli.co.jp）を **新しいPCで一から始める／作業を再開する**ための手順です。

## 大前提（どこに何があるか）
- **本当の保管庫 = GitHub リポジトリ**（`https://github.com/sakita2026/applipplikeyweb`）。
  `commit → push` したものは全部クラウド(GitHub)にあり、PCが壊れても失われません。
- **公開先 = Azure**（`https://key.applippli.co.jp`）。`main` に push すると GitHub Actions が自動デプロイします。
- **localhost:8080 = 各PC内のプレビュー**。保存対象ではなく、下記でいつでも作り直せます。
- ⚠️ 共有フォルダ（Azure File Share）では作業しない（git リポジトリでなく assets/.github が欠けており、push するとサイトが壊れます）。

## 新しいPCでのセットアップ
1. **git** と **Node.js** をインストール
   - git: https://git-scm.com/ ／ Node.js: https://nodejs.org/
2. リポジトリを取得（初回は GitHub 認証を求められます＝ブラウザでログイン）
   ```
   git clone https://github.com/sakita2026/applipplikeyweb.git
   cd applipplikeyweb
   ```
3. プレビューを起動
   ```
   node dev/serve.js
   ```
   ブラウザで **http://localhost:8080/** を開く。
   同じWi-Fiのスマホからは **http://（このPCのIPアドレス）:8080/**（IPは `ipconfig` で確認）。

## 日々の作業フロー
```
git pull origin main            # 始める前に必ず最新化
（index.html / help.html を編集）
git add index.html              # 自分が変更したファイルだけ（-A は使わない）
git commit -m "変更内容"
git push origin main            # push すると自動デプロイ
```
- push が弾かれたら `git pull origin main` してから再 push。
- `git push --force` は使わない（相手の変更が消えます）。
- こまめに commit & push する＝それがバックアップになります。

## 2名共同編集のルール
- 有田・先田の2名で index.html / help.html を編集。
- 各自が自分のPCに clone し、自分の localhost でプレビュー（localhost は共有できません）。
- 同じファイルの同じ箇所を同時に触らない（「今 index 触ります」と一声）。
- 大改修時はブランチ＋Pull Request。

## デプロイ確認
push 後、GitHub Actions が完了すると `https://key.applippli.co.jp/` に反映されます。
